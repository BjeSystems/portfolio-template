# Workflow: handoff и возвращение к проекту

## Идея

Сохраняйте состояние проекта в git и в файле handoff. При следующем открытии Cursor вы продолжаете с того же места, без необходимости помнить все детали.

## Handoff (HANDOFF-NEXT-CHAT.md)

В корне проекта создайте файл с кратким контекстом для нового чата:

- Что уже сделано (контент, контакты, фото, репо, деплой).
- Что осталось по желанию (правки, кастомный домен).
- Промпт для агента: «Прочитай HANDOFF-NEXT-CHAT.md и продолжи. Ответы — по-русски» (или на своём языке).

При возвращении к проекту откройте handoff и вставьте этот промпт в новый чат — агент подхватит контекст.

## Возвращение к проекту

1. Откройте папку проекта в Cursor.
2. Откройте новый чат с агентом.
3. Вставьте промпт из handoff или напишите: «прочитай handoff и продолжи».
4. Добавьте одну фразу: что хотите сделать сегодня (деплой, правка текста, фото и т.д.).
5. Дальше действуйте по шагам с агентом.

Состояние уже сохранено в файлах и в коммите — этого достаточно. Остальное по желанию и по силам.
