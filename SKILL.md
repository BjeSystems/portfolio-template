# Skill: одностраничное портфолио (визитка)

## Когда применять

- Пользователь просит создать портфолио/визитку на одной странице.
- Пользователь хочет подставить контакты, фото или текст в уже существующий шаблон портфолио.
- Пользователь просит задеплоить статическую визитку (например на Vercel).

## Контекст шаблона

- Шаблон лежит в репозитории **portfolio-template**: один `index.html` (стили внутри), папка `img/` для фото, опционально handoff и docs.
- Палитра по умолчанию — тёмная («салун»): фон #252a33, текст крем/янтарь. Фото в круге с рамкой и лёгкой тонировкой.
- Контакты: один или два email, GitHub, Telegram. В шаблоне — плейсхолдеры (email@example.com, github/username, t.me/username).

## Алгоритм

1. **Данные:** Узнать у пользователя (если ещё не указано): имя для заголовка и alt фото; email(ы); GitHub (URL или username); Telegram (URL или username). Опционально: подзаголовок, краткое описание, текст блока «Обо мне», URL фото или локальный путь к файлу.
2. **Файлы:** Открыть `index.html` в проекте пользователя (или скопировать шаблон из portfolio-template в новый каталог). Заменить плейсхолдеры: `<title>`, `<h1>`, `.hero-title`, `.hero-desc`, блок `<section id="about">`, ссылки в `<section id="contacts">` (mailto:, href на GitHub и Telegram), `alt` у изображения.
3. **Фото:** Если пользователь дал URL — подставить в `src` (учтите ограничения referrer у некоторых CDN). Предпочтительно: пользователь кладёт файл локально (например `img/hero.jpg`), в HTML указывается `src="img/hero.jpg"`. При необходимости подстроить `object-position` и `transform: scale()` для кадрирования в круге.
4. **Визуал (по запросу):** Менять только по явной просьбе: цвета фона/текста/рамки в `<style>`, размер круга (`.hero-photo-wrap`), эффекты на фото (filter, overlay).
5. **Git и деплой (если просят):** В каталоге проекта: `git init -b main`, `git add .`, `git commit -m "Initial commit: portfolio"`. Создать репозиторий на GitHub (по умолчанию приватный, если иное не сказано), добавить remote, push. Деплой: `vercel --prod` из корня или импорт репо в Vercel Dashboard. После деплоя можно зафиксировать в README или handoff прод-URL.
6. **Чувствительные данные:** Перед пушем в **публичный** репозиторий проверить, что в файлах нет паролей, токенов, личных email/имён, если пользователь не хочет их светить. В публичном шаблоне использовать только плейсхолдеры.

## Плейсхолдеры в шаблоне

- Имя: «Ваше имя» или «Имя».
- Email: `email@example.com`.
- GitHub: `https://github.com/username`.
- Telegram: `https://t.me/username`.
- Текст «Обо мне»: короткий универсальный абзац-заглушка; пользователь заменяет своим.

## Заметки

- Статика: сборка не нужна, Vercel отдаёт файлы как есть.
- Handoff: если в проекте есть HANDOFF-NEXT-CHAT.md, обновлять его после деплоя (указать прод-URL и убрать пункты «осталось» про push/deploy).
